Django AB-Testing

What is this lib about?

AB Testing is very specific, but the basics are same: you prepare two ore more experiments to measure the success.

The lib takes care for managing and providing tests, goals, results and experiments.

Implementation

I think we can divide implementations into two groups:

  1. switches in templates

    Changes for tests are implemented in the 'live' template. Obviously the template needs to know about the current experiments, to render the correct experiment. Advantage of this method is fast implementation and you have everything at one place; but you should implement default handlers for your switches, if tests become inactive and experiments are not longer provided to the template.

  2. one template per experiment

    You do not touch your live templates, but create a new (probably derived) template per Experiment. In this scenario the view needs to know about the current active experiments.

The lib allows you to mix both methods as you like, by shipping a middleware that choose and store experiments per session. They are available in request.abTest; together with a context processor, that make these information available to templates.
